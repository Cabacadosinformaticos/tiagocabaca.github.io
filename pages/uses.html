O meu setup
de desenvolvimento

Hardware, software e ferramentas que uso no dia-a-dia. Atualizado regularmente.

// Hardware

💻
Gigabyte AERO 15 (2021)

O portátil principal. Ecrã de 144Hz, bom desempenho para desenvolvimento e para correr servidores locais sem stress.

🖥️
PC Desktop

Máquina de secretária mais antiga, usada para tarefas mais pesadas em casa.

🖥️
AOC C27G2ZU/BK 27" 240Hz FHD — Monitor Principal

Monitor curvo de 27 polegadas a 240Hz. A taxa de refresh alta faz diferença no dia-a-dia, mesmo fora de jogos.

🖥️
AOC 25" 144Hz + Monitor Vertical

Monitor secundário de 25" a 144Hz para conteúdo de apoio, e um terceiro monitor em formato vertical — ideal para ler grandes blocos de código no editor sem fazer scroll constantemente.

⌨️
Logitech MX Keys S (portátil) · Logitech G512 (desktop)

O MX Keys S para trabalho e mobilidade — silencioso e confortável para escrever durante horas. O G512 no desktop para quando quero mais feedback tátil.

🖱️
Logitech MX Master 3S (portátil) · Logitech G502 (desktop)

O MX Master 3S tem o scroll eletromagnético que não consigo mais dispensar. O G502 no desktop para precisão e peso personalizável.

🎧
7Hz x Crinacle Zero:2 (IEM)

Mudei de headphones para IEM e não volto atrás. Os Zero:2 têm uma resposta em frequência muito equilibrada para o preço, excelentes para música e para isolar do ambiente.

// Editor & Terminal

✏️
VS Code

O editor principal. Uso um tema escuro com a fonte JetBrains Mono. Extensível o suficiente para qualquer stack, rápido o suficiente para não me irritar.

code.visualstudio.com →
📦
Extensões VS Code

ESLint, Prettier, GitLens, Error Lens, Tailwind CSS IntelliSense, Auto Rename Tag, REST Client.

⚡
Windows Terminal

Terminal no Windows com Git Bash e PowerShell. Perfis separados para cada contexto de trabalho.

🐙
Git + GitHub

Controlo de versões em linha de comandos. Todos os projetos no GitHub — público quando faz sentido.

github.com/Cabacadosinformaticos →

// Stack preferida

⚡
JavaScript / TypeScript + React + Vite

A combinação que uso na maioria dos projetos frontend. TypeScript evita uma classe inteira de bugs, e o Vite torna o desenvolvimento muito mais fluido.

🟢
Node.js + Express

Para backends e APIs REST. Simples, rápido de montar e com um ecossistema enorme. A escolha certa para a maioria dos projetos web.

🐍
Python + FastAPI

Para quando preciso de algo mais próximo do sistema, automações, ou APIs onde Python faz mais sentido. FastAPI é surpreendentemente rápido e a tipagem com Pydantic ajuda muito.

🗄️
MySQL / SQLite

MySQL para projetos com mais volume e concorrência, SQLite para ferramentas locais e protótipos rápidos. Bases de dados relacionais primeiro — NoSQL quando o problema pede.

🎨
Tailwind CSS

Para styling rápido e consistente. Demora um dia a habituar, depois não voltas ao CSS puro para projetos grandes.

tailwindcss.com →

// Apps & Produtividade

🖼️
Figma

Para wireframes, protótipos e design de interfaces antes de passar a código. Também uso para diagramas rápidos.

figma.com →
📝
Notion

Notas, documentação de projetos e organização pessoal. A flexibilidade das bases de dados do Notion é difícil de substituir.

📷
Lightroom / Darktable

Para edição das fotos. Lightroom quando preciso de algo rápido, Darktable quando quero mais controlo — e é open source.

🌐
Chrome DevTools

A ferramenta que mais uso no desenvolvimento frontend. Performance, rede, responsividade — está tudo ali.
